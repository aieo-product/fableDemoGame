# Fable Katamari — ブラウザ3Dゲーム（Fable 5 性能デモ）

塊魂インスパイアのブラウザ3Dゲーム。ボールを転がしてオブジェクトを取り込み、どんどん大きくなる。
成長すると「一段階大きい世界」へ**シームレス**に移行し、画鋲→車→ビル→**月**とスケールが5桁変わっても60fpsを維持する設計。

**Play**: https://fable-katamari.pages.dev

## 操作

- **WASD / 矢印キー**: 転がす
- **Space**: ダッシュ（ゲージ式）/ **Shift**: ブースト / **M**: ミュート
- **マウスドラッグ**: カメラ回転（任意）
- **目標**: 大きくなって**東京スカイツリー**を巻き込め（タイムアタック、クリアタイムでS〜Dランク判定）

## v5 ポリッシュアップデート

- **開幕の完全リニューアル**: スタート店は実在の秋葉原・千石電商の位置をテーマにした「センゴク電子」。スポーン周囲にパーツの輪+出口へ続くパーツの道+🔩誘導矢印で、転がして2秒以内に最初の吸収
- **スタックチャン**: コレクション13個目としてM5Stackロボ「スタックチャン」が店の棚に（ドナックのいとこ）
- **アキバらしい建物**: ゲームセンター・家電量販店・メイドカフェ・PCパーツビルが電気街に登場
- **宇宙エンディング**: スカイツリーを巻き込むと地球から離陸 — 眼下に夜の地球、日本列島には金色の街明かりが瞬く
- **修正**: 開幕不可視バグ（実在の総武線高架がスポーン直上に被っていた）、道路リボンの「謎の溝」（実半径0.6〜3mでフェードイン）

## v4 リアル東京アップデート（OpenStreetMap）

- **実データの東京ストリート**: 秋葉原アンカー半径2.5km（実寸）の詳細ディスク+渋谷・浅草パッチに、OpenStreetMap 由来の**実在ビル 14,563棟**を実座標（水平1:5・高さ1:2.5）でボクセル化して配置。すべて巻き込み可能なゲームオブジェクト
- **実道路・河川・公園**: 道路/鉄道リボン・神田川/隅田川・皇居・上野公園などの実ジオメトリ地表レイヤー（+2ドローコール）
- **モデル品質パス**: ヒーロー12種の高品質化（三角形上限350→600）、頂点AOベイク、空色リムライト（各々キルスイッチ付き）
- **データレース対策**: タイル取得がティア2到達に間に合わなければ恒久的に手続き生成へフォールバック（ワンウェイラッチ、`?osmdelay=ms` でテスト可能）

### 地図データのライセンス（ODbL）

- 地図データ © [OpenStreetMap contributors](https://www.openstreetmap.org/copyright)、ライセンス: [Open Database License (ODbL) 1.0](https://opendatacommons.org/licenses/odbl/)
- **抽出日時**: 2026-06-11（`public/assets/tokyo/tokyo-v4-manifest.json` の `extractionDate` が正本）
- **派生データベースの提供**: 本リポジトリの [`data/osm-raw/`](data/osm-raw/)（タグ削減済み生Overpassレスポンス 7.3MB/124ファイル）と [`scripts/osm/`](scripts/osm/)（再現可能な変換パイプライン）が ODbL の derivative database 公開義務を構成する
- **パイプライン**: `npm run osm:fetch`（再開可能なOverpass取得）→ `npm run osm:build`（決定論的変換、再実行でバイト同一）→ `npm run osm:verify`（予算/カウント/カバレッジ/ナビゲビリティ検証 — `predeploy` ゲートに組込み済み）
- **データ予算**: 出荷バイナリ合計 ~284 KB gz（core 105KB + outer 181KB、ハードキャップ 1,536 KB）

## v3 箱庭東京アップデート

- **箱庭東京マップ**: 秋葉原のパーツショップ（2cmの玉でネジや抵抗を巻き込む）から始まり、中央通り→上野・浅草・丸の内・渋谷の各街区→**東京スカイツリー(634m)を巻き込んでゴール**。雷門・西郷さん像・東京駅・109・国会議事堂・東京ドーム・レインボーブリッジ・東京タワーなど11のランドマークを実寸スケールで配置
- **スマホ最適化**: モバイルファーストHUD（セーフエリア対応、全ビューポートで重なりゼロ検証済み）、タッチジョイスティック+ダッシュボタン
- **巻き込み名表示**: 「+120 ネジ」のように何を巻き込んだか常時フィードバック（連続巻き込みはマージ表示）
- **レアコレクション**: 金の招き猫・ハチ公像など12種の収集アイテム。巻き込むと実物サムネイル付きポップアップ、リザルトに4x3のコレクション図鑑（周回を跨いでlocalStorage保存）
- **ドナック実況**: 公式キャラ「ドナック」（緑帽子のアヒル）が吹き出しでランドマーク豆知識・プレイのコツ・お祝いコメント（優先度+クールダウン制で邪魔しない設計、OFFトグルあり）

## v2 月アップデート

- **月ゴール**: 序盤から空に見える月が、500m到達で降臨→接触→夜空への昇天エンディング
- **BGM**: WebAudio完全合成のボサポップ（128BPM、成長でレイヤー解放、ミュート可）
- **背景強化**: 太陽/月/星/流れる雲のスカイシェーダ、山並み⇔スカイラインの遠景シルエット
- **巻き込めない大物**: 鳥居・五重塔・城・鉄塔・スカイツリー・山など12種のランドマーク
- **スコアアタック**: コンボ、きらめくレアアイテム、タイムボーナス、自己ベスト保存
- **リザルトをXでシェア**: タイム/スコア/ランクを #FableKatamari 付きでポスト

## 技術ハイライト

- **Three.js r177 + Vite 6 / 純粋静的サイト**（サーバーレス、Cloudflare Pages）
- **相似変換リスケール**: シミュレーションは常にボール半径0.5〜2.5の数値帯で実行。ティア昇格時に全座標へ S=0.2 の一様スケールを1フレームで適用し、カメラがスケール量の純関数であるため描画はピクセル同一＝遷移は完全に不可視。実寸は double で別管理（Float32精度を無限成長まで保証）
- **シームレス法則**: 吸収判定・カメラ・フォグ・速度はすべて半径の連続関数。ティア番号は見た目（スポーン内容・パレット・演出）のみを駆動 → 閾値でのポップ/ヒッチが構造的に存在しない
- **自作アーケード物理**（約400行、依存ゼロ）: 動体はボール1個のみ。ティア帯別の空間ハッシュ（フラット型付き配列）で衝突候補をマイクロ秒で取得
- **インスタンス描画**: アーキタイプ×ティアの InstancedMesh プール、ドローコール上限55。取り込んだオブジェクトはボール子階層に取り付け時1回だけ行列書き込み、埋没したら自動間引き
- **決定論的チャンクスポーン**: シード付きPRNGによる手続き配置、フレームあたり≦64スポーンに償却、フォグの外で出現
- **アセットゼロ**: 全ジオメトリ・色・効果音（WebAudio合成）をコードで生成

## 開発

```bash
npm install
npm run dev      # 開発サーバー
npm run build    # dist/ にビルド
```

デバッグ: バッククォートキーでパフォーマンスオーバーレイ、`?r=<メートル>` で任意サイズから開始、`?seed=<n>` でワールド再現。

## 制作について

本プロジェクトは Claude (Fable 5) が ultracode マルチエージェントワークフローで制作:
設計（3視点並列提案+審査統合）→ 実装（契約凍結+5並列ストリーム+統合）→ レビュー（多次元+敵対的検証）→ デプロイ。
詳細は作業ログ参照。
