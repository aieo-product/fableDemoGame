**主要な意思決定**:

- Both OSM pools register ALL 16 voxel geometries; pool membership is enforced by BAND routing (osmPoolForBand: band<=3 -> detail, else large), not per-code partition — a record's band derives from r_eff, so any code can legally appear in either pool; cost is a few KB of duplicated unit-box vertex data, zero extra draws.
- Ground tile ring-exit HIDES instances via setVisibleAt instead of deleting geometry: BatchedMesh space cannot be compacted without optimize() stalls, and the all-resident worst case (measured ~72.7k verts / ~38.6k tris on shipped bins) must fit anyway, so capacity is whole-map and hidden instances cost zero draws/tris under multi-draw.
- OsmGround decodes the ground sections itself from raw outer-shard bytes (osmWorld.outerBytes contract member) rather than having OsmWorld pre-decode roads/polys — keeps the building/tower decode in Stream W and the render-only tessellation in Stream R with a one-field interface.
- OSM water surface y baked at 0.03 game m — between parks (0.02) and minor roads (0.04): ponds/moats render over park patches, roads bridge over rivers; offsets scale with the similarity so the rescale invariant holds.
- Road ribbons tessellate as independent segment quads with square caps (each end extended by halfWidth) — covers outer-corner gaps of butt joints; overlaps are same-color same-depth so they read as one ribbon; max ROAD n=31 keeps the scratch buffers module-static (zero per-frame alloc).
