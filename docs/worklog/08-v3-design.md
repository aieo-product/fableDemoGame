# Phase 9 作業ログ: v3設計（ワークフロー `katamari-v3-design`）

**要件（ユーザー指示）**: スマホ表示最適化（UI被り解消）/ 箱庭東京マップ（秋葉原パーツショップ開始→街→ランドマーク→スカイツリーゴール）/ 巻き込み名表示 / レアコレクション+リザルト一覧+X自慢 / ドナック吹き出し実況 / Xポスト修正（先行ホットフィックス済み: twitter.com/intent/tweet + text/url/hashtags分離方式）

**ゴールの曖昧性解決**: ユーザーは「スカイツリーがゴール」と「東京タワーを巻き込むまで」の両方に言及 → 東京タワー(333m)は終盤ランドマーク、スカイツリー(634m)が最終ゴールとして両立させる解釈を採用（明示的なゴール変更指示の方を優先）。

**事前準備**: ドナックのキャラクター仕様を公式リポジトリ(donack_characterc_design)から調査（緑帽子+白羽毛+黄嘴のピクセルアートアヒル、表情4種スプライト、明るく簡潔な相棒口調）。表情フレームを120x90 WebPに前処理。旧・銅像設計の批評15件（屋内カメラ破綻/ドアソフトロック等）をサルベージして設計入力に使用。

**手法**: 設計→敵対的批評→改訂の3段。実行実績: エージェント3体 / 約459,716トークン / 約37分

## 批評が検出した主要問題（改訂で解決）

- [blocker] **Curated landmarks/collectibles are invisible to absorb/pushback outside their tier-band window — ball rolls straight through Tokyo Tower**
- [blocker] **'Skytree visible from frame one' is physically impossible (CAMERA_FAR=4000 sim ≈ 160m real at boot) AND the draft deletes the v2 sky→mesh handoff it needs as fallback**
- [blocker] **Phase-0 MOON_*→GOAL_* renames break the build for every stream — the plan's own exit criterion ('boots with zero stream files touched') is unsatisfiable as written**
- [major] **Knocked-off curated EXTRA objects silently vanish: spawner.reinject returns false for unknown archetype ids, contradicting the spec's 'goes through the existing chunk reinject path'**
- [major] **Pacing spec contradicts itself by 3×: tier table claims ~6:00–6:30 optimal while tuning.js comments claim optimal 120–135 sim-s with RANK_S_S=150 — and v2 empirical data says the constants deliver ~100s, not 6 minutes**
- [major] **Shop shelf-top items (incl. 2 of the 4 shop collectibles) are unreachable within the no-seal budget, and terrain release leaves elevated placements floating in mid-air**
- [major] **Draw-call ledger's '≤10 active curated-extra' is unenforced and optimistic, and the ledger miscounts the bay (water quad + quay strip = 2, counted as 1)**
- [major] **Skytree base is pushback-free for the whole run — the ball rolls through the 634m goal monument's legs from T0 to T6**
- [major] **Ball start position and ?at= teleports are unspecified against BallPhysics.reset's hardcoded (0,r,0) — salvaged critique #14 is only half-resolved**
- [major] **Donack asset pipeline is broken on this machine and ignores already-shipped assets: pngquant is not installed, and 26 webp frames (96KB, 120×90) already exist in public/assets/donack/**
- [major] **FOG_FAR_MIN_M=9m violates the fog<loadRadius invariant outdoors at r<0.16m — pop-in on the Akiba street right after the shop exit**
- [minor] **Mobile HUD spec gaps: timer slot arithmetic doesn't fit its own font spec, landscape toast anchor is unspecified, and the dash-ring --gauge animates in 10Hz steps**
- [minor] **ハチ公像 is simultaneously a landmark and collectible #11 — double event, double popup, and a Donack priority collision the trigger system doesn't define**
- [minor] **Collection bitmask schema has no append/reorder rule — a v3.1 content patch silently corrupts every player's album**
- [minor] **T1→T2 exit bridge is unverified: a player leaving the shop at r 0.10–0.25m faces street content authored for 0.5m+ with only an unquantified pigeon/cat sprinkle in between**
- [minor] **Curated EXTRA pool slots and the attach animation: render/ball steals instanceSlot during the ABSORB emit, but slot return/ownership for curated-owned pools is unspecified**
- [minor] **Donack idle/edge checks subscribe to a 1Hz GrowEvent-fed poll, but GrowEvent stops during the finale and overlays — unspecified suppression windows risk bubbles over the cinematic**

## 改訂時の意思決定

1. BLOCKER 1 (curated objects invisible to absorb/pushback outside band window) — RESOLVED via DYNAMIC RE-BANDING, not the alwaysLive-array alternative: on activation and on every EVT.TIER_UP, curated re-stamps store.tierOf of every alive curated slot to clamp(naturalBand, tierIndex-1, tierIndex+1), amortized inside the existing 64/frame round-robin; hashes already rebuild on tier change so re-banded slots enter the live hash set for free. Chosen over the alwaysLive array because the failure class covers all ~370 curated placements (a T0 ball must bounce off a band-2 vending machine on the street), not just the 24 landmarks/collectibles. DEV assert in curated.update: every alive curated slot's band is inside the live window.

2. BLOCKER 2 (Skytree 'visible from frame one' impossible at CAMERA_FAR=4000 sim) — RESOLVED by making the v2 moon handoff the spec instead of deleting it: environment.js KEEPS the sky-element shader slot re-textured as a Skytree silhouette (azimuth recomputed per frame from SKYTREE_POS), active while worldScale < SKY_SILHOUETTE_WS_MAX (0.2); render/goalTower.js mesh takes over with the proven v2 crossfade at the first frame tower simDist < 0.8*CAMERA_FAR. Uniforms RENAMED (uMoon*->uGoalSil*), never removed. CAMERA_FAR stays 4000 (no depth-precision gamble). Stream A test matrix gains a ?at=shop boot screenshot asserting the silhouette is on screen at r=0.02.

3. BLOCKER 3 (MOON_*->GOAL_* rename breaks boot for every stream; 15 files verified by grep) — RESOLVED with Phase-0 deprecated aliases: events.js ships EVT.MOON_CALL = EVT.GOAL_CALL (etc., same string values so subscriptions are identical objects) and tuning.js re-exports all MOON_* names (@deprecated) including MOON_GOAL_RADIUS_M = GOAL_RADIUS_M, MOON_RADIUS_K, MOON_DESCENT_S, MOON_LAND_*. Each stream deletes its own alias consumers as it lands; the integrator deletes the alias block last with a hard gate: grep -rn 'MOON_' src returns zero. Phase-0 exit criterion re-stated: npm run dev boots WITH aliases present and zero stream files touched.

4. MAJOR 4 (knocked-off curated EXTRA objects vanish: spawner.js:452 returns false for codes >= 70) — RESOLVED: render/ball.knockOff SKIPS stuck entries with archetype code >= 70; landmarks and collectibles are permanently stuck once absorbed (thematically correct — you do not bonk Tokyo Tower off the ball; collection/landmark credit was granted at absorb). DEV assert in spawner.reinject that archIdx >= 0 for everything reaching it. Curated placements that use chunk archetypes (codes < 70) DO reinject via the existing chunk path, stripping FLAG_CURATED|FLAG_RARE. The misleading 'goes through the existing chunk reinject path' blanket sentence is deleted.

5. MAJOR 5 (pacing contradiction 6min vs 130s) — RESOLVED by picking ONE pacing truth in Phase 0 and deleting the losing numbers: GROWTH_K=10 physics kept (absorb feel + Tokyo Tower jump math depend on it); the owner's 5-8min band is met by AUTHORED density, not physics: chunk zone density = DENSITY_K_V3 (0.45) of v2 per-band counts, finite-map travel legs added. Binding targets: typical first clear 5:30-6:30, optimal/speedrun ~3:30-4:00 sim-s. RANK_S_S=240/A=330/B=450/C=600, TIME_BONUS_FULL_S=240/ZERO_S=600 (EMPIRICAL Phase-3 retune >=3 playthroughs remains the verification, not the design). Shop budget derived and validator-asserted: ~240 interior placements (avg r~2.2cm, SigmaR^3~2.6e-3 m^3) support full-clear to r~0.30m; tier table dwell column rewritten to the same truth (45/40/45/45/50/55/35s + travel). The draft's 60/50/55... dwell table and RANK_S_S=150 are DELETED so no implementer anchors on them.

6. MAJOR 6 (shelf-top items unreachable within no-seal budget; floating items after terrain release) — RESOLVED with three data-level fixes: (1) interior item heights capped at y <= 0.7 (shelf mid); collectibles 金の招き猫 and 真空管 move to the low table (y 0.4), レトロゲーム機 to the floor at shelf B's end, 秋葉原フィギュア to shelf A mid y 0.7 (reachable from r~0.33, validated); validateCityMap() adds the per-placement 3D-reach inequality sqrt(dXZ^2+(y-r)^2) <= 1.45r+objR with dXZ >= r at the placement's intended tier radius; (2) on SHOP_TERRAIN_RELEASE curated lerps every still-alive elevated placement's py to restY = r*(1+yOff) over the same 0.6s as the mesh zero-scale (amortized, <= 240 slots) — items drop as shelves dissolve; (3) tier table re-labeled so elevated content is honestly late-T1.

7. MAJOR 7 (draw-call ledger optimistic + bay miscount) — RESOLVED by making the contingency the design: EXTRA archetypes render from 4 SHARED InstancedPools partitioned by size class (collectible-small cap 12, landmark-mid cap 4, landmark-large cap 4, landmark-XL cap 4 incl. shop shell), per-instance color; worst case flat +4. Honest ledger: 40 world + 8 stuck + 6 fixed + 1 backdrop + 2 skytree + 1 terrain + 2 water+quay + 4 extra = 64; DRAW_CALL_CAP=72 with real headroom. Ambiguity resolved in one sentence: collectibles/landmarks DO ring-deactivate (consume=false, identity preserved by the consumed bitmask) when outside loadRadius and reactivate on approach; only SIZE-based despawn is forbidden for them.

8. MAJOR 8 (ball rolls through Skytree base all game) — RESOLVED: CityTerrain gains one permanent static collider — circle-vs-ball XZ test at SKYTREE_POS, radius SKYTREE_BASE_R_M*SKYTREE_COLLIDER_K (90*0.6), real-meter converted like shop walls, emits BOUNCE, never absorbs, NOT released at 4.0m. Finale contact uses PAD 0.85 > 0.6 so the finale always triggers before the collider; that inequality is asserted in validateCityMap().

9. MAJOR 9 (ball start vs hardcoded reset(0,r,0); ?at= teleports unspecified) — RESOLVED structurally: cityMap is authored with ORIGIN = BALL START; the shop interior geometry is re-published in ball-origin coordinates (shop center at (+1.6,-0.8)); district/landmark coordinates keep their published values (the 1.6m shift is noise at district scale and is absorbed into the authoring). BallPhysics.reset stays untouched. devTeleport(name) is specced into the Phase-0 main.js skeleton: snap worldScale to 0.04*5^k with k minimal s.t. r/ws in [0.5,2.5); set ball pos/radiusSim; spawner.onTeleport() (band-cache invalidation + preload); curated.forceScan() (full-placement pass ignoring the 64 budget); terrain release re-eval; one forced maybeRebase. Integrator-owned, ~15 lines, written down before integration week. While re-deriving I found the draft's start position was INSIDE shelf row B's prism footprint — the re-published layout fixes it and the validator now asserts ball-start clearance.
