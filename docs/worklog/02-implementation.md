# Phase 2 作業ログ: 実装（ワークフロー `katamari-implement`）

**手法**: 契約凍結 → 5並列開発ストリーム → 統合の3段構成。リードエージェントが型定義・設定・コアユーティリティを先に確定し、5エージェントが互いのファイルに触れない分担で並列実装、最後に統合エージェントがmain.jsを配線してビルド+ブラウザ実機スモークテストまで実施。

**実行実績**: エージェント7体 / サブエージェント消費 約1,029,579トークン / 所要 約65分

**結果**: ビルド成功 / 実行時スモーク合格 / バンドル 153.4 KB (gzip)

## Stage 1: 契約凍結（リード）

**作成ファイル**: `index.html`, `public/_headers`, `src/types.js`, `src/config/tiers.js`, `src/config/tuning.js`, `src/core/events.js`, `src/core/rng.js`, `src/core/pool.js`, `src/core/mathUtils.js`, `src/main.js`

**実装アプローチ**:

Implemented Phase 0 strictly from docs/DESIGN.md. src/types.js centralizes every cross-module JSDoc typedef (Tier, Archetype, BallState, Intent, StuckRecord, WorldReentry, and all 10 event payload shapes) with `export {}` to make it importable, zero runtime code. src/config/tuning.js exports ~70 heavily-commented named constants covering every feel number in the design (ACCEL_K=22, SPEED_K=8.5, ABSORB_RATIO=0.65, GROWTH_K=0.45, CAM_DIST_K=6.5/CAM_HEIGHT_K=3.2, FOG 14/55, budgets 64, stuck cap 512, burial 0.98, sub-pixel 0.04, sim band 0.5..2.5, start 0.05m, win 500m, etc.). src/config/tiers.js freezes the 6-tier table (Desk/Room/Street/Town/City/Skyline, enterTrueRadius 0.05/0.25/1.25/6/30/150, cellSizeSim 32, loadRadiusSim 96, objectsPerChunk 72, 8 archetype ids per tier = 48 unique frozen ids, per-tier fog/sky hex palettes) plus RESCALE_S=0.2 and DEV-guarded invariant asserts (6 tiers, monotonic radii, 48 unique ids, 1/RESCALE_S === SIM_RADIUS_MAX/SIM_RADIUS_MIN, FOG_FAR_K < loadRadiusSim - cellSizeSim). src/core/events.js is a Map-of-arrays EventBus with indexed-loop zero-alloc emit, frozen EVT name constants, pre-allocated PAYLOADS objects for every event, and a shared `bus` singleton. src/core/rng.js has mulberry32, an imul-avalanche hash(seed,cx,cz,tier), and resolveWorldSeed (?seed= or Date.now). src/core/pool.js has FreeList (LIFO Int32Array stack, allocs come out 0,1,2..., -1 on exhaustion) and IntRing (power-of-two FIFO over Int32Array, push returns false when full, shift/peek return -1 when empty). src/core/mathUtils.js has analytic critically-damped springScalar/springVec3 (mutate-in-place, zero alloc), underdamped springDamped for the ball y-pop, damp(halflife), lerp/clamp/easings, formatLength + splitLength({value,unit} reuse), and shared TMP_V3A/B/C, TMP_QA/QB, TMP_M4 scratch from three. src/main.js is a runnable skeleton: GameState machine (title/playing/win) driven by bus events, fixed-60Hz accumulator with MAX_SUBSTEPS=3 and debt-drop, the exact 7-step per-frame call order from the design as structured comments with commented stub calls, all Phase-1 module imports pre-written but commented, and clearly-marked TEMPORARY start/restart button wiring. index.html provides the canvas + full HUD/title/win/debug DOM skeleton with frozen ids, bilingual JA/EN labels, and embedded dark-theme CSS (glassy corner panels, gradient title, tier banner pop animation). public/_headers sets immutable caching for /assets/* and must-revalidate for HTML. Everything passed node --check and a runtime smoke test that also manually exercised the tiers.js invariants (skipped in node because import.meta.env is undefined there).

**主要な意思決定**:

- Frozen the 48 archetype ids in tiers.js (design only gave examples for T0/T2/T3/T5): T1 Room = book/mug/shoe/plushie/lamp/cat/stool/trash_can, T4 City = apartment/crane/billboard/water_tower/smokestack/warehouse/overpass/gas_tank, T5 filled to 8 with radio_tower/dome/cruise_ship. All lowercase snake_case — Dev E's catalog.js MUST implement exactly these spellings (asserted unique=48 in dev).
- objectsPerChunk = 72 for ALL tiers, matching the design's explicit '~72/chunk at load radius 96' math (pi*96^2/32^2 ~ 28 chunks * 72 ~ 2030 alive = the 2000 target budget). N+1 scenery band overrides (radius 140, 8/chunk) live in tuning.js as SCENERY_LOAD_RADIUS_SIM / SCENERY_OBJECTS_PER_CHUNK because they are band-relative, not per-tier.
- Event payloads are pre-allocated in events.js as a single exported PAYLOADS object (PAYLOADS.absorb, PAYLOADS.grow, ...) plus frozen EVT name constants — emitters mutate then emit, so zero-alloc emit is structural, not a convention each dev re-invents.
- Tuning constants are NAMED EXPORTS (not one default object) so modules can import exactly what they use and `import * as TUNING` still works; names like CAM_POS_OMEGA/CAM_LOOK_OMEGA (6.0/4.5) and BALL_Y_OMEGA/BALL_Y_ZETA (16/0.6) freeze the spring vocabulary.
- mathUtils provides BOTH analytic critically-damped springs (springScalar/springVec3 for camera — never overshoots, framerate-independent) and an underdamped springDamped(omega, zeta) for the ball ground-y 'pop' which the design explicitly wants to overshoot.
- IntRing documents the value domain as non-negative int32 (returns -1 for empty) and rejects push when full instead of overwriting — dropping spawn/despawn queue entries silently would desync wanted-vs-loaded chunk state; callers retry next frame.
- tiers.js fog invariant assert uses reference simRadius 1.0 (FOG_FAR_K 55 < loadRadiusSim 96 - cellSizeSim 32 = 64), matching the design's literal '55r < 96-unit load radius' phrasing.
- main.js includes minimal TEMPORARY start/restart button listeners (clearly marked 'REMOVE when ui/screens.js is integrated') so the skeleton is demonstrably alive in a browser during Phase 1; everything else is commented stub calls in the binding 7-step frame order.
- vite.config.js left untouched: it already has base './' and target es2022; Vite 6 with a single entry and no dynamic imports emits a single JS chunk by default, satisfying the design without config churn.
- Dev asserts guarded with `import.meta.env && import.meta.env.DEV` (not optional chaining on env directly) so the files also load cleanly under plain node for headless module tests.

**設計からの逸脱**:

- Design's tier table says 'x5/tier' but lists T3 entering at 6m (1.25*5=6.25). I kept the design's literal 6m since tierIndex is cosmetic-only and decoupled from the simRadius-2.5 rescale; consequence: tierUp T3 fires slightly before the corresponding rescale. Harmless by the seamlessness law.
- Design typedef sketch for events.js implied payload reuse but gave no concrete mechanism; I froze it as the exported PAYLOADS object + EVT constants (additive, not contradictory).
- WorldReentry (return of ball.knockOff) had no field spec in the design; froze it as { archetypeId:string, radiusSim:number, pos:Vector3, vel:Vector3 } in types.js — ball.js (Dev D) must keep a per-slot archetypeId record to populate it, and spawner (Dev B) re-injects from it.
- vite.config.js not rewritten (already conformant).

## Stage 2: 並列開発ストリーム

### ストリーム A: シミュレーションデータ+物理

**作成ファイル**: `src/world/objects.js`, `src/world/spatialHash.js`, `src/physics/ballPhysics.js`, `src/physics/absorb.js`

**実装アプローチ**:

ObjectStore is a pure SoA over typed arrays (px/py/pz/radius F32, archetype U16, tierOf/flags U8, instanceSlot I32) with a FreeList allocator, an aliveCount counter, idempotent free() guarded by the flags byte, and branchless full-capacity passes for rescaleAll(S) and rebaseAll(dx,dz); it also hosts the archetype code<->id mapping (code = tier*8 + indexInTier) derived at module load from the frozen TIERS table. SpatialHash implements the spec's counting-sort design: a packed entries Int32Array grouped by hashed cell key with cellStart[TABLE+1] ranges, imul-based key ((xi*PX)^(zi*PZ))&16383, tombstone (-1) removes with opportunistic repack past TOMBSTONE_REBUILD_FRAC, plus a 256-entry loose buffer for post-rebuild O(1) inserts that an internal allocation-free double-buffered repack folds back in when full; queryBall dedups hash-colliding cells via a per-instance visited-key scratch and key-matches the loose buffer, guaranteeing a superset of true overlaps. BallPhysics owns BallState and integrates camera-relative accel (documented yaw convention: yaw about +Y, 0 looks -Z), per-frame friction pow, radius-proportional speed cap, rolling-without-slipping via premultiplied axis-angle quaternion on module scratch, an underdamped ground y-spring (springDamped) whose hidden state is covered by rescale(S)/rebase/reset hooks for ScaleManager, and linear sluggishness recovery sized so one absorb's dip heals in exactly SLUGGISH_RECOVERY_S. Absorb.resolve runs per fixed substep: combo/bonk timers, visual radius slew (<=RADIUS_SLEW_K*r/s), then queryBall on each of the 3 band hashes into a 1024-entry Int32 scratch; per candidate it does a 3D sphere overlap with collisionScale fudge, dispatching to cbrt volume growth + hash.remove + 'absorb' emit (before store.free so render/ball can read fields synchronously) or to a planar-XZ pushback whose separation target sqrt(sum^2-dy^2) makes the corrected 3D distance exact, with restitution reflection, thresholded 'bounce' emit, and cooldown-gated 'knockOff' (count from a fixed-seed mulberry32). Headless-verified: brute-force superset/staleness/dup checks on the hash through insert/remove/repack/rebuild cycles, physics direction/cap/spring/quat invariants, growth math and event payload fields, and a post-GC steady-state loop showing ~0 B/step allocation at ~0.55 microseconds per step+resolve.

**主要な意思決定**:

- SpatialHash adds a small (256) loose insert buffer on top of the spec's counting-sort packed arrays: packed arrays physically cannot take O(1) inserts, and the spawner inserts up to 64/frame continuously; the loose buffer is folded in by an allocation-free internal repack (cached keys, double-buffered entries) every ~4 frames worst case (~0.02ms), and queryBall key-matches it so correctness is preserved.
- Archetype Uint16 encoding (code = tier*8 + indexInTier, derived from frozen TIERS) lives in objects.js as ARCHETYPE_ID_BY_CODE/ARCHETYPE_CODE_BY_ID/archetypeCode() — the store owns its field's encoding, absorb resolves event archetypeId strings from it without depending on Dev E's catalog.js, and spawner gets a single import site.
- Absorb emits 'absorb' BEFORE store.free(i): render/ball's attachStuck needs px/py/pz/radius/instanceSlot intact during the synchronous emit to start the attach animation; the typedef's 'already freed; do not deref next frame' wording stays satisfied since free happens immediately after dispatch.
- Pushback uses a planar XZ normal with separation target sqrt(sum^2 - dy^2) instead of a raw 3D normal: the ball is ground-constrained (can only move in XZ), and this makes the corrected 3D center distance exactly equal the radius sum even for elevated bounding spheres (buildings), avoiding jitter against tall objects.
- BallPhysics keeps vel strictly horizontal (vel.y === 0) and drives pos.y from the hidden underdamped spring; rescale(S)/rebase(dx,dz)/reset(r) hooks scale/shift that hidden spring state too, so ScaleManager must call ballPhys.rescale(S) rather than mutating state fields directly (otherwise the y-spring pops on the rescale frame, breaking pixel-identity).
- queryBall callers pass maxObjR = hash.cellSize * 0.5 (16 sim units): a safe over-bound for any object in a band without tracking per-band max radii; with cell 32 this still scans only 2x2-3x3 cells.
- Knock-off count (1-3) comes from a fixed-seed mulberry32, and bonks are gated by a 0.25s cooldown so one wall scrape across 3 substeps emits exactly one knockOff; bounce events below 5% of speedCap are suppressed as resting contact.
- Sluggishness recovers linearly at (1-SLUGGISH_FACTOR)/SLUGGISH_RECOVERY_S per second so a single absorb's 0.03 dip heals in exactly 1.5s and rapid absorbs stack proportionally deeper — matches the 'recovering over 1.5s' spec without an exponential tail that never completes.

**設計からの逸脱**:

- SpatialHash: added a 256-entry loose insert buffer + internal repack alongside the specified counting-sort packed arrays — the design's 'insert once at spawn' and 'counting-sort packed entries' are mutually exclusive without it (packed arrays have no O(1) insert). Queries remain exact-superset; cost is a bounded loose scan per query.
- Absorb constructor extended to (bus, scaleProvider?, catalog?) versus main.js's 'new Absorb(bus)' sketch: worldScale is required for trueRadius/sizeReal event fields, and collisionScale comes from CATALOG (read once at construction, defaults to 1 when absent so headless tests work).
- Added two local constants not in tuning.js: BOUNCE_EMIT_MIN_FRAC = 0.05 (suppresses resting-contact bounce event spam) and BONK_COOLDOWN_S = 0.25 (one knockOff per bonk, not one per substep). Both are sound-minimal guards; promote to tuning.js if the designer wants them tweakable.
- Pushback normal is planar (XZ) with a 3D-consistent separation target rather than the raw 3D contact normal — the ball cannot move in Y, so a 3D normal would leave residual penetration against elevated bounding spheres.
- objects.js exports archetype code mapping helpers and ObjectStore.rebaseAll(dx,dz); ballPhysics exports rescale/rebase/reset hooks — additive API beyond the interface sketch, needed by ScaleManager/spawner and flagged below.

### ストリーム B: ワールド（スポーン/スケール管理）

**作成ファイル**: `src/world/spawner.js`, `src/world/scaleManager.js`

**実装アプローチ**:

Spawner: each tier band lives on its own global chunk grid in that band's NATIVE sim units (worldScale_b = 0.1*5^b); conversion to current units is an exact 5^(band-k) where k = rescale count, so chunk coordinates and consumed bitmasks are invariant under both rescale and rebase (origin shift is accumulated as originX/Z in current units, *=S on EVT.RESCALE, +=shift on onRebase). Chunk keys pack band|cx|cz into one int31 for Map keys. Wanted-vs-loaded diffing runs only when the ball crosses a chunk boundary per band (cached bx/bz) and covers bands N (loadRadiusSim 96 native, 72/chunk), N+1 scenery (SCENERY_LOAD_RADIUS_SIM 140 in the scenery tier's native units, 8/chunk — matches the ~600 alive budget), and N+2 pre-warm at PREWARM_FRACTION (capped to 1200 current units, placements inside the fog wall skipped so giants never pop into view; they stay out of the hashes until the mandatory tier-up rebuild). Placement j of a chunk is O(1)-regenerable: per-placement sub-seed hash(chunkSeed, j, 0x5bd1, band) drives an inline (closure-free) mulberry32 with a FIXED draw order, position comes from a 37-strided 9x9 jittered sub-grid. Spawn/despawn are amortized through preallocated IntRings (<=64 each/frame, front-of-motion via insertion-sorted new chunks), with generation counters on both chunk records and store slots to invalidate stale queue entries; chunk records (512 preallocated) carry packed (storeIdx|placement) entries for ring-exit fade-outs. Density top-ups handle the scenery->target role upgrade at tier handoff (placements [enqueuedThrough..72) enqueue), N-1 leftovers are never re-spawned but unload via a round-robin distance scan, bands <=N-2 stop being wanted at tier change and their bitmasks are forgotten. Sub-pixel sweep scans 200 slots/frame round-robin (kill cap 32, marked consumed). Knock-off WorldReentry records fly on a 16-slot ballistic pool and insert into the hash on landing. ScaleManager: maybeTierUp applies the one-frame similarity rescale at simRadius>=2.5 (ball pos/vel/radii *= S, store.rescaleAll, pool.rewriteAll(S), cameraRig.rescaleState(S), optional env.onRescale(S), then synchronous EVT.RESCALE so spawner/render-ball ride along), derives tierIndex from trueRadius with TIER_HYSTERESIS independently (T3 enters at 6 m vs rescale at 6.25 m), rebuilds the three banded hashes (hashes[i] = tier-1+i) when either fires, emits throttled 10 Hz 'grow' with progress01ToNextTier, and latches 'game:win' with the seed. maybeRebase integer-snaps (Math.round) the ball position out of everything past 1500 sim units (store typed-array loops, pool.rebaseAll, cameraRig.rebaseState, spawner.onRebase) and rebuilds hashes. A 40-assertion headless smoke test (fake store/hash/pool against the frozen contracts) verified budgets, bit-exact deterministic regeneration, consumed-bitmask persistence across unload/reload, tier-up+rescale invariants, pre-warm, sub-pixel drain, rebase, reinjection, win latch and reset.

**主要な意思決定**:

- Tier-band native coordinates: SCENERY_LOAD_RADIUS_SIM (140) is interpreted in the SCENERY tier's native sim units (not current units) — this is the only reading that reproduces the design's ~600-alive/75-chunk scenery budget; target-band loadRadiusSim is native==current by definition.
- N+2 pre-warm ring is additionally capped at 1200 CURRENT sim units (a full 140-native(+2) ring would be 3500 current units, violating the +-2048 coordinate bound); pre-warm placements inside the fog wall are skipped so future-tier giants never pop into view — the small post-tier-up hole is invisible and refills on chunk reload.
- Rescale and tierIndex change are detected independently inside maybeTierUp (simRadius>=2.5 vs trueRadius>=enterTrueRadius with hysteresis) because TIERS T3 enters at 6 m while x5 growth rescales at 6.25 m; hashes rebuild if EITHER fires, normally the same frame.
- Spawner learns about rescales via the synchronous EVT.RESCALE emission (maybeTierUp's frozen signature has no spawner param); native chunk coords are provably invariant under rescale ((x+origin)*=0.2 while k++), so only originX/Z, invWorldScale, the exponent and in-flight reentries are touched.
- Consumption is tracked via EVT.ABSORB subscription + per-slot Int32 chunkKeyOf / Int16 placementOf side arrays (spawner-owned, not in the SoA contract); 8-bit generation counters on store slots and chunk records invalidate stale ring-queue entries after slot recycling — no scanning, no allocation.
- Per-placement O(1) regeneration: instead of replaying the chunk's single rng stream (O(j) skips), each placement gets sub-seed hash(chunkSeed, j, 0x5bd1, band) feeding an inline mulberry32 step (no closure allocation per spawn); draw order is documented as the determinism contract.
- N-1 leftovers have no wanted set (never re-spawned, never mass-deleted): they drain via absorb + sub-pixel sweep + a 16-records/frame round-robin distance cleanup at loadRadius+cell current units, matching the 'leftovers falling' budget curve.
- Despawn frees store+hash immediately and delegates the visual scale-fade (and instance-slot reclaim) to pool.fadeOut — objects at the despawn ring are beyond fog so sim-instant removal is invisible; this avoids a spawner-side per-object fade ticker.
- ScaleManager emits 'grow' (10 Hz via performance.now throttle, first emission immediate) and the once-latched 'game:win' with worldSeed taken from its constructor — it is the single owner of trueRadius-derived events.
- InstancedPool similarity hooks are called as rewriteAll(S) (uniform scalar: matrix elements [0..14] *= S) and rebaseAll(sx,sz) (translation-only), guarded with typeof checks + DEV warnings so partial integration cannot crash.
- store.alloc() is assumed to return a bare index; the spawner writes all SoA fields directly (px/py/pz/radius/archetype/tierOf/flags=1/instanceSlot) per the public-field contract, and defensively zeroes flags after free.
- Numeric archetype convention frozen and exported: ObjectStore.archetype U16 = tier*8 + positionInTier derived from TIERS[t].archetypeIds; spawner.js exports ARCHETYPE_IDS / archetypeIndexFor / archetypeIdFor for absorb.js and others.
- CATALOG is taken as an optional 6th constructor arg (not imported) so spawner.js never hard-fails if config/catalog.js hasn't landed; missing entries get neutral per-tier placeholder stats and the pool lookup is lazy (pools may be created after Spawner construction).

**設計からの逸脱**:

- Spawner.update takes two extra params beyond the main.js stub sketch: update(ballPos, tierIndex, ballRadiusSim, dt) — ball radius is required by the continuous fog/sub-pixel/pre-warm rules and dt by knock-off flight integration (both default safely).
- ScaleManager constructor is (bus, worldSeed=0), one arg more than the 'new ScaleManager(bus)' sketch, because it emits 'game:win' which must carry the seed; main.js should pass worldSeed.
- InstancedPool.rewriteAll is called with the scalar S (uniform similarity: elements [0..14] *= S) rather than a callback as the DESIGN.md sketch 'rewriteAll(fn)' hints; an additional rebaseAll(sx, sz) pool method is REQUIRED for floating-origin rebase (translation-only rewrite) — neither existed as a precise contract, both are guarded and dev-warned if absent.
- N+2 pre-warm is capped at 1200 current sim units and skips placements inside the fog wall (see keyDecisions) — a literal full-radius pre-warm would break the +-2048 coordinate bound.
- Ring-exit despawns free the store slot and remove from the hash immediately (visual fade only, handled by the pool) instead of fading in-sim for 0.4 s — despawning chunks are beyond the fog wall so this is unobservable and removes a per-object fade ticker from the spawner.
- Knock-off ballistic gravity is a local constant (REENTRY_GRAVITY = 30 sim/s^2) since tuning.js has no such constant; trivially movable to tuning.js in a tuning pass.
- tierIndex can flip slightly before the rescale (T3 at 6 m vs rescale at 6.25 m) because the tier table is not an exact x5 ladder; both events independently force a hash re-band/rebuild so nothing desyncs (cosmetic-only divergence, per the seamlessness law).

### ストリーム C: 描画コア

**作成ファイル**: `src/render/renderer.js`, `src/render/geometryFactory.js`, `src/render/instances.js`, `src/render/environment.js`

**実装アプローチ**:

renderer.js: Renderer class owns the WebGLRenderer (antialias, high-performance), the THREE.Scene, and the PerspectiveCamera (FOV_BASE, near 0.05, far 4000). Pixel ratio = min(dpr, PIXEL_RATIO_MAX=1.5); the dynamic-resolution governor accumulates frame intervals into a GOVERNOR_WINDOW_S (3s) window and on window close steps the pixel ratio down 0.125 toward 1.0 if avg > FRAME_BUDGET_MS (17ms), or back up toward base if avg < 0.7x budget (hysteresis prevents oscillation). Backquote toggles the #debug-overlay (fps/draw calls vs cap/tris vs budget/alive/pixel ratio/heap delta, 5Hz text refresh while visible); KeyR (DEV only) fires a settable onForceRescale callback; parseStartRadius() parses the ?r= dev param. Emits EVT.FRAME_STATS via the reused PAYLOADS.frameStats after each render. geometryFactory.js: buildAllGeometries(CATALOG) calls each archetype's buildGeometry with a deterministic mulberry32(hash(GEOMETRY_SEED, fnv1a(id), 0, tier)) rng, ensures normals + vertex colors (white fallback + dev warn), normalizes every geometry to bounding sphere center=origin/radius=1 (binding contract for spawn scaling), and dev-asserts the 350-tri cap; also exports mergeColoredParts (tint-then-mergeGeometries helper for catalog recipes), bakeVertexColors, normalizeToUnitRadius, triangleCount, disposeGeometries. instances.js: InstancedPool stores per-slot TRS in flat Float32Arrays (source of truth), composes matrices scalar-wise straight into instanceMatrix.array (verified bit-equal to Matrix4.compose), uses FreeList slots with zero-scale kill and high-water-mark count, eager DynamicDrawUsage instanceColor, scale-fade animator (compact swap-remove active list; fadeIn ease-out cubic, fadeOut cubic shrink that AUTO-FREES the slot at completion), min/max dirty-range tracking flushed as exactly one needsUpdate + r177 clearUpdateRanges/addUpdateRange per attribute per frame, rewriteAll(fn) plus a tight-loop rescaleAll(S) for the one-frame similarity rescale, and reset() for game reset; getSharedObjectMaterial() is the single MeshLambertMaterial({vertexColors:true}) singleton, updateAndFlushPools(pools, dt) is the main.js step-7 helper. environment.js: Environment builds the 4096x4096 ground plane with a ShaderMaterial procedural grid (cell = 2*radiusVisualSim, fwidth-antialiased minor+5x major lines, custom linear fog uniforms replicating THREE.Fog exactly so ground fogs identically to Lambert instances), a camera-following sky gradient dome (direction-only gradient blending to fog color at the horizon, BackSide, depthWrite off, renderOrder -10), scene fog at FOG_NEAR_K/FOG_FAR_K * radiusVisualSim, hemisphere + directional lights (no shadow maps, light tracks ball at constant direction), and a canvas radial-gradient blob shadow decal (1.15r). Tier palettes (fog/skyTop/skyBottom + derived ground/line/hemi colors per tier, precomputed at boot) crossfade over PALETTE_FADE_S on the 'tierUp' bus event by lerpColors into live Color objects that the fog/uniforms/lights reference directly. rescale(S) scales the accumulated grid-phase origin (plus radius-derived values defensively); rebase(dx,dz) shifts the grid origin so the floating-origin rebase never visibly jumps the grid. All per-frame paths are zero-allocation; everything is recomputed from ball state each update so the rescale frame is pixel-identical. Headless-tested with node (48 assertions: normalization, determinism, matrix compose equality, update-range exactness, fade lifecycle, auto-free, rescale similarity, exhaustion, reset).

**主要な意思決定**:

- GEOMETRY NORMALIZATION CONTRACT: buildAllGeometries() recenters + uniformly rescales every archetype geometry to boundingSphere center=(0,0,0), radius=1.0. WHY: decouples catalog modeling units from spawn scaling — an instance with bounding radius r sim units is exactly setTransform(pos, quat, scale=r) with pos.y = r*(1+yOffset). Dev B (spawner) and Dev D (ball) MUST scale by radius, not by catalog units.
- InstancedPool.fadeOut(slot, s) AUTO-FREES the slot when the fade completes (zero-scale + free-list reclaim). WHY: the design says despawn = store/hash removal at despawn time + 0.4s visual fade; making the pool own fade-end reclaim means the spawner can forget the slot immediately with no callback plumbing.
- Added InstancedPool.update(dt) beyond the frozen method list. WHY: the scale-fade animator needs a per-frame step; folding it into flush() would hide a dt parameter the frozen signature doesn't have. updateAndFlushPools(pools, dt) wraps both for main.js step 7.
- Added rescaleAll(S) alongside the frozen rewriteAll(fn). WHY: the standard similarity transform (pos*=S, scale*=S, recompose) is a single tight typed-array loop; rewriteAll(fn) remains for arbitrary rewrites (e.g. floating-origin rebase via fn calling setTransform).
- Governor recovers resolution (step up 0.125 when 3s avg < 0.7x budget) in addition to the specified drop-toward-1.0. WHY: a transient spike would otherwise permanently degrade resolution; the 17ms-drop / 11.9ms-recover hysteresis gap prevents oscillation.
- Ground fog uses custom uniforms that replicate THREE.Fog's LINEAR factor exactly (not smoothstep). WHY: the ground must fog identically to the MeshLambert instances or the fog wall would show a seam between objects and the plane they sit on.
- Environment tracks a uGridOrigin uniform: rescale(S) multiplies it by S, rebase(dx,dz) adds the subtracted offset. WHY: the grid pattern is the only accumulated (non-radius-derived) visual state; without this the floating-origin rebase would visibly snap the ground grid under the ball.
- Sky dome follows the CAMERA (not the ball) when a camera is provided. WHY: the gradient is direction-only, so camera-following makes the rescale frame exactly pixel-identical (ball-following leaves a ~1% gradient parallax shift that the force-rescale screenshot diff could catch).
- Environment subscribes to EVT.TIER_UP and EVT.GAME_RESET on the imported singleton bus (constructor signature stays new Environment(scene, camera?) per the main.js sketch). Force-rescale KeyR is DEV-gated; backquote overlay works in prod too (it is advertised on the title screen).
- Eager instanceColor creation (white) in the InstancedPool constructor. WHY: lazy creation on first setColorAt would recompile the shader program mid-game; eager creation keeps exactly one program from frame zero.

**設計からの逸脱**:

- Renderer constructor also creates and owns THREE.Scene and the PerspectiveCamera (design table only says 'WebGLRenderer setup') — matches the main.js sketch which references a `scene` and `renderer.camera` but never constructs them elsewhere.
- Environment constructor takes an optional second `camera` argument (main.js sketch shows `new Environment(scene)`); pass renderer.camera for exact rescale pixel-identity of the sky dome. Falls back to ball-follow if omitted.
- InstancedPool grew three methods beyond the frozen list — update(dt) (fade animator step), rescaleAll(S) (tight similarity loop), reset() (game reset) — all additive; the frozen surface (alloc/free/setTransform/setColor/fadeIn/fadeOut/rewriteAll/flush) is implemented exactly.
- Dynamic-resolution governor includes a hysteresis recovery step back toward the base pixel ratio (design only specifies dropping toward 1.0).

### ストリーム D: フィール（入力/カメラ/ボール/演出/音）

**作成ファイル**: `src/input/input.js`, `src/render/cameraRig.js`, `src/render/ball.js`, `src/render/effects.js`, `src/audio/sfx.js`

**実装アプローチ**:

Input aggregates keyboard (WASD/arrows, Shift=boost), a window-anchored touch joystick (first touch = anchor, 64px radius, deadzone 0.15, second touch = boost), and primary-button mouse drags into one reused Intent object; mouse drag accumulates a yaw delta drained via takeYawDrag(). CameraRig runs two critically-damped Vector3 springs (springVec3 from mathUtils) in sim space for position (target = ballPos - fwd*6.5r + up*3.2r) and look (ballPos + vel*0.4), with lazy shortest-angle yaw-follow of velocity (0.45s halflife), a recentering mouse yaw offset, FOV = 60 + sin-envelope tierUp kick to 68 over 0.8s (with the position spring omega dipped 40% for the overshoot breath) + damped speed bonus above 80% cap, and exponential-decay bonk shake driven off EVT.BOUNCE; rescaleState(S) multiplies all spring state and the live camera position, and a rebase(offset) hook handles floating-origin shifts. Ball builds a noise-displaced icosphere core (MeshLambertMaterial + onBeforeCompile injecting uLump displacement and a uColorA/uColorB swirl mix, colors lerped per absorb by BALL_COLOR_LERP) plus 8 ball-parented stuck InstancedMeshes of 64 slots each (512-record power-of-two ring with generation counters); attachStuck computes the write-once ball-local socket (invQuat*dir*r*0.92 divided by group scale) and queues a 0.15s ease-out squash animation through an IntRing of packed gen<<9|idx entries (falling back to an immediate final write if the queue saturates); a staggered burial sweep covers all 512 records per BURIAL_STAGGER_S culling buried/sub-pixel records to zero-scale matrices via per-family FreeLists, with min/max dirty-slot tracking flushed through addUpdateRange once per mesh per frame; knockOff(n, ball) walks the ring backward from newest, fills 3 preallocated WorldReentry records (archetypeId resolved from a flat tier*8+slot table built from TIERS), and rescaleState(S) scales group.scale plus record radii so burial math stays in current sim units. Effects runs three small additive instanced QuadPools (pops 96, sparkle ring 64, speed lines 32) with SoA Float32Array particle state and overwrite-oldest cursors; bus handlers only queue bursts which materialize at the ball surface in update(dt, ballState), with EVT.RESCALE scaling live particle state by S. Sfx synthesizes everything from oscillators plus an LCG-noise roll loop (lowpass+gain following setRollIntensity(speed01)), creating/resuming the AudioContext on first window gesture; absorb blips rise one semitone per combo, clonk volume follows impactSpeed01, tierUp plays a C-E-G triangle arpeggio, win a 5-note fanfare. Validated headlessly with node against mocked SoA stores and BallState: ring cap/steal behavior, burial drain to zero, knockOff reentries, spring snap distance, yaw convergence, FOV kick peak/return, rescale/rebase identity, and exact socket world position at group scale 0.04.

**主要な意思決定**:

- Stuck pools use 8 generic low-poly PROXY geometries (box/sphere/cylinder/cone/slab/pillar/ring/gem, unit bounding radius) tinted per-instance — an InstancedMesh has exactly one geometry, so 48 real archetype shapes cannot fit the frozen 8-stuck-mesh draw budget; family = archetypeCode & 7
- Assumed ObjectStore archetype Uint16 code = flat catalog index tier*8 + slotInTier in the order frozen in tiers.js archetypeIds — knockOff resolves archetypeId strings from a table built from TIERS, with zero dependency on catalog.js (Dev E)
- Attach animation runs inside the stuck pool (world->ball-local start lerped to socket) instead of animating the world InstancedPool slot — keeps Ball fully decoupled from Dev C's instances.js while preserving the 0.15s squash money shot
- Ball record bookkeeping radii (attachRadiusSim/objHalfSim) are kept in CURRENT sim units and multiplied by S in ball.rescaleState(S), so the burial test compares directly against currentSimRadius with no per-frame unit conversion
- Anim-queue saturation fallback: if the packed-gen IntRing push fails, the record is finalized live immediately — found via smoke test (88 records were permanently stuck in 'animating' and leaked their slots)
- Generation counters (22-bit, packed gen<<9|idx) on ring records make stale anim-queue entries self-invalidating after ring wrap or knock-off culls
- CameraRig/Effects/Sfx subscribe to the singleton bus in their constructors (eventBus param defaults to bus) so tierUp/bounce/absorb feel works with zero extra wiring; CameraRig also self-snaps on GAME_START/GAME_RESET to avoid spring lag pop-in
- Speed-FOV bonus and speed lines key off FOV_SPEED_FRAC of the continuous speed cap SPEED_K*radiusSim — never tierIndex (seamlessness law)
- Effects subscribes to EVT.RESCALE to scale live particle world positions by S (Effects is not passed to ScaleManager.maybeTierUp, and the event is emitted synchronously in the same between-update-and-render slot)
- Sfx roll loop gets speed via an explicit setRollIntensity(speed01) method because no bus event carries per-frame speed; silent no-op if never wired
- Mouse yaw drag is captured in Input (per the file-table spec) but applied by CameraRig via an explicit yawDrag parameter — degrades safely to no-op if main.js never forwards it

**設計からの逸脱**:

- Ball.update takes (dt, ballState) instead of the main.js stub's ball.update(frameDt) — Ball must sync the group transform and run the burial test against currentSimRadius; main.js must pass ballPhys.state
- Effects.update takes (dt, ballState) instead of effects.update(dt) — absorb/tierUp payloads carry no position, so bursts materialize at the ball surface using the passed state; bursts stay harmlessly queued if omitted
- ball.knockOff signature is knockOff(n, ballState) rather than knockOff(n) — ejection origin/velocity need ball truth and absorb.js has it in hand
- Stuck instances render proxy shapes, not the absorbed archetype's real merged geometry (hard consequence of one-geometry-per-InstancedMesh x 8-mesh budget); the real shape is visible until the absorb moment, and the squash attach pop masks the swap
- Added ball.rescaleState(S) and cameraRig.rebase(offset)/ball-record scaling beyond the spec'd cameraRig.rescaleState(S): ScaleManager's frame-order signature receives cameraRig but not ball, yet ballGroup.scale *= S and record-radius scaling must happen somewhere — ScaleManager (or main) MUST call ball.rescaleState(S) during the rescale or stuck objects will not ride the similarity transform
- attachStuck accepts an optional 4th param colorHex (the absorbed instance's tint) — AbsorbEvent carries no color; falls back to a built-in per-family palette when omitted

### ストリーム E: コンテンツ+UI（48アーキタイプ/HUD/画面）

**作成ファイル**: `src/config/catalog.js`, `src/ui/hud.js`, `src/ui/screens.js`

**実装アプローチ**:

catalog.js: All 48 frozen archetype ids implemented as data objects in a CATALOG record, each with a buildGeometry(rng) recipe composed from boot-time helpers (box/cyl/cone/sph/ico/torus + a towerBanded helper that bakes alternating wall/window vertex-color rows with rng-lit windows onto a single height-segmented BoxGeometry — used by apartment and skyscraper). Each part is transformed (scale->rotate->translate) then painted with a baked vertex-color attribute (flat or vertical gradient) in linear working space via THREE.Color.setHex; parts are flattened to non-indexed (Icosahedron is non-indexed, merge requires consistency) and merged with mergeGeometries, then recentered on the bounding-sphere center and normalized to an exact unit bounding sphere. Verified headlessly with a node script driving all 48 builders: every geometry is <=268 tris (cap 350), bounding radius exactly 1.0, has a color attribute; total 5,878 tris across the whole catalog. yOffset values were measured from the normalized geometry (yOffset = -1 - minY so the visual bottom touches the ground at restY = radius*(1+yOffset)) and written back, so grounding is exact, not eyeballed. Dev-mode assert block cross-checks the catalog against TIERS (48 ids present, tier/id field match, palette 4-6 entries, sane ranges). hud.js: Hud class subscribes once to GROW/ABSORB/TIER_UP/GAME_START/GAME_WIN/GAME_RESET with prebound methods; all DOM writes go through diffed writers (cached last string/number, assign only on change) gated at HUD_THROTTLE_HZ via performance.now(); the absorbed counter stores a pending count that the (already 10Hz) grow event flushes so a burst's final value always lands; unit rollover pops #size-unit via a transform kick the existing CSS transition eases back; tier banner adds/removes 'show' on a timer; splitLength reuses one out object. screens.js: Screens emits game:start/game:reset from the buttons but drives ALL overlay visibility from bus events (start hides title+win, reset shows title, win fills #win-size via formatLength and #win-seed then reveals), so debug-key starts keep the DOM consistent. Both UI classes were smoke-tested headlessly with a stub document/performance: 24/24 assertions pass (throttling, unit rollover, trailing-count flush, banner, win/reset flow, seed fallback).

**主要な意思決定**:

- Geometry normalization contract: every buildGeometry output is recentered and scaled to an EXACT unit bounding sphere (radius 1.0, origin = bounding center), so instance scale = the placed object's radiusSim directly — this makes spawner/instances math trivial and collisionScale meaningful against the true bounding radius.
- yOffset convention concretized as restY = radius * (1 + yOffset) (matches types.js '0 = sphere sits on ground'); values were MEASURED from the built geometry (-1 - minY) via a headless script and hard-coded back, so objects rest exactly on y=0.
- All parts are converted toNonIndexed() before mergeGeometries because IcosahedronGeometry is non-indexed while Box/Cylinder/Sphere are indexed — mergeGeometries hard-fails on mixed indexing (found by the headless test on 'tree').
- Tint strategy: archetypes with a colorful 'body' bake that part near-white and carry saturated palettes (car, mug, book, hydrant...); fixed parts (wheels, metal, glass) are baked dark/desaturated so instanceColor multiplication only nudges them — keeps 4-6-tint variety without destroying readability.
- Window bands on apartment/skyscraper are vertex-colored row bands on ONE height-segmented BoxGeometry (towerBanded helper) with rng-driven lit windows — per the design's 'vertex-color window bands', costing ~60-70 tris instead of extra boxes.
- Hud: absorbed counter throttles its own writes but stores a pending count that the 10Hz grow event flushes — guarantees the trailing value of an absorb burst is rendered without any per-frame tick or rAF (hud stays purely event-driven per main.js's 'hud is event-driven').
- Screens: clicks only EMIT; overlay visibility reacts to bus events (game:start hides title+win, game:reset shows title, game:win reveals win) — keeps DOM consistent even when game:start is emitted by a debug key instead of the button.
- Restart flow: ROLL AGAIN emits game:reset and returns to the TITLE screen (it does not auto-start), matching main.js's state machine where GAME_START is accepted from TITLE or WIN.
- Hud also subscribes to GAME_START/GAME_WIN/GAME_RESET to own #hud visibility and reset its readouts — main.js does not need to touch HUD DOM at all.

**設計からの逸脱**:

- A few landmark archetypes use 7-11 primitives instead of the suggested 2-6 (ferris_wheel 11, bridge_span 9, crane 7, water_tower 6+) where charm required it (gondolas, suspension cables, splayed legs); all stay well under the 350-tri cap (max is ferris_wheel at 268).
- die uses 5 pip boxes (4 small dark boxes on two faces) rather than literal pip geometry on all faces — readability at game scale with 60 tris.
- Hud applies its own 10Hz gate on top of the emitter-side 10Hz 'grow' throttle (belt-and-suspenders; worst case the HUD updates at ~5-10Hz, visually indistinguishable).
